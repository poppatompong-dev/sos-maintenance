# 💾 พกพาใส่แฟลชไดรฟ์ (ทำงานต่อแบบไม่ต้องมีเน็ต)

ทางเลือกนอกจาก git/GitHub — ถ้าอยากคัดลอกทั้งโปรเจกต์ใส่ USB แล้วไปทำต่อที่เครื่อง
อื่น (แม้เครื่องนั้นไม่มีอินเทอร์เน็ต)

> เครื่องปลายทางยังต้องมี **Node 22 + pnpm** ติดตั้งไว้ (ติดตั้งครั้งเดียว มีตัวติดตั้งแบบ
> ออฟไลน์) และต้องมี **Docker Desktop** เฉพาะตอนทำงานฐานข้อมูล — แต่ **เปิดดู UI /
> รันเทสต์ ไม่ต้องใช้เน็ตและไม่ต้องใช้ Docker**

## สร้างชุดพกพา (ทำที่เครื่องนี้ ตอนที่ยังมีเน็ต)

```powershell
cd C:\dev\sos-maintenance

# แบบพร้อมใช้ออฟไลน์ (รวม node_modules ให้เลย — แนะนำสำหรับ USB)
pwsh ./scripts/make-portable.ps1 -Offline

# หรือแบบเล็ก (เฉพาะซอร์ส ต้อง pnpm install ที่ปลายทางโดยมีเน็ตครั้งเดียว)
pwsh ./scripts/make-portable.ps1
```

ได้โฟลเดอร์ **`C:\dev\sos-maintenance-portable`** → **ลากทั้งโฟลเดอร์ใส่แฟลชไดรฟ์**

โฟลเดอร์นี้รวม: ซอร์สโค้ด + เอกสาร + `.git` (ประวัติครบ, push กลับ GitHub ได้ภายหลัง) +
`node_modules` (ถ้าใช้ `-Offline`). ไม่รวมของที่สร้างใหม่ได้ (`.next`) และไม่รวม `.env`
ตัวจริง

## ใช้งานที่เครื่องปลายทาง

```powershell
# 1. คัดลอกโฟลเดอร์จาก USB ลงเครื่อง (อย่ารันจากบน USB โดยตรง จะช้า)
#    เช่น คัดลอก sos-maintenance-portable -> C:\dev\sos-maintenance
cd C:\dev\sos-maintenance

# 2. สร้าง .env จากตัวอย่าง
Copy-Item .env.example .env

# 3a. ถ้าสร้างแบบ -Offline มาแล้ว: รันได้เลย ไม่ต้องมีเน็ต
pnpm dev            # http://localhost:3000  และ  /today
pnpm test           # 182 tests

# 3b. ถ้าสร้างแบบเล็ก: ต้อง install ก่อน (ครั้งเดียว ต้องมีเน็ต)
pnpm install
pnpm dev
```

## ซิงก์กลับ / ทำงานสลับ USB ↔ GitHub

โฟลเดอร์พกพารวม `.git` ไว้แล้ว → เมื่อเครื่องปลายทางมีเน็ต ทำงานร่วมกับ GitHub ได้ปกติ:
```powershell
git pull            # ดึงงานล่าสุด (ถ้ามี)
# ... ทำงาน ...
git add -A && git commit -m "..." && git push
```
ถ้าไม่มีเน็ตเลย: ทำงาน + `git commit` บน USB ไปก่อน แล้วค่อย `git push` ทีหลังเมื่อเจอเน็ต
หรือคัดลอกโฟลเดอร์กลับมาก็ได้

## ข้อควรรู้
- `node_modules` ที่รวมมาเป็นของ **Windows x64** — ใช้ได้กับเครื่อง Windows 64-bit
  ด้วยกัน ถ้าย้ายข้ามระบบปฏิบัติการ ให้ลบ `node_modules` แล้ว `pnpm install` ใหม่
- แฟลชไดรฟ์ควรเป็น **exFAT/NTFS** (FAT32 มีขีดจำกัดไฟล์ 4GB — โปรเจกต์นี้ไม่ถึง แต่
  exFAT ปลอดภัยกว่า)
- `.env` ไม่ถูกคัดลอก (กันความลับหลุด) — สร้างใหม่จาก `.env.example` ที่ปลายทาง
